# WAgent多AI协同智能写作系统 - Web前端要求文档
文档版本：V1.0
更新日期：2026年4月18日
关联文档：《WAgent多AI协同智能写作系统 项目文档（完整版）》

## 一、前端概述
### 1.1 定位与目标
Web前端是WAgent系统的核心交互界面之一，与本地桌面端功能完全对齐，为用户提供远程、跨设备的小说创作体验。前端需实现会话管理、多AI协同创作交互、小说内容查看/修改/下载等核心功能，确保操作流畅、视觉美观、数据与后端实时同步。

### 1.2 核心设计原则
- **功能一致性**：与本地桌面端功能100%对齐，会话、创作数据多端实时同步。
- **交互直观性**：采用左中右三区经典布局，操作路径清晰，降低用户学习成本。
- **视觉现代化**：支持深色/浅色主题切换，界面简洁美观，反馈及时。
- **响应式适配**：适配桌面端、平板端等不同屏幕尺寸，确保核心功能可用。

## 二、技术栈要求
| 技术/库 | 版本要求 | 用途 |
|---------|----------|------|
| Vue | 3.4+ | 核心前端框架，采用Composition API开发 |
| Vue Router | 4.2+ | 路由管理，实现会话列表与创作界面的切换 |
| Pinia | 2.1+ | 状态管理，存储会话、对话、设置等全局状态 |
| Axios | 1.7+ | HTTP客户端，封装与后端FastAPI的通信 |
| Element Plus | 2.6+ | UI组件库，提供对话框、按钮、列表等基础组件 |
| @vueuse/core | 10.9+ | Vue组合式工具库，简化本地存储、响应式变量等操作 |
| markdown-it | 14.0+ | Markdown渲染库，用于预览小说章节内容 |
| highlight.js | 11.9+ | 代码高亮库（可选，用于增强Markdown预览效果） |

## 三、功能需求详细说明
### 3.1 整体页面结构（左中右三区布局）
页面采用固定的左中右三栏布局，各区域功能独立且协同，整体布局不随页面切换改变。

| 区域 | 宽度占比 | 固定/可调整 | 核心功能 |
|------|----------|-------------|----------|
| 左侧会话选择区 | 20% | 固定（最小宽度240px） | 会话列表展示、新建/切换/删除/搜索会话 |
| 中间对话框区 | 50% | 固定（最小宽度600px） | AI输出展示、用户输入、创作流程交互 |
| 右侧小说内容区 | 30% | 固定（最小宽度320px） | 章节列表展示、章节内容查看/修改/下载 |

### 3.2 左侧会话选择区
#### 3.2.1 功能清单
- 会话列表展示
- 新建会话
- 切换会话
- 删除会话
- 搜索会话
- 会话状态标识

#### 3.2.2 详细功能描述
1. **会话列表展示**
   - 列表项显示内容：会话绑定的**小说名称**（主标题，字号14px，加粗）、最后更新时间（副标题，字号12px，灰色）。
   - 排序规则：默认按最后更新时间倒序排列，最新的会话在最上方。
   - 选中状态：当前选中的会话列表项高亮显示（背景色使用主题色的10%透明度）。
   - 空状态：无会话时，显示“暂无会话，点击上方按钮创建”提示文字。

2. **新建会话**
   - 入口：列表顶部的“新建会话”按钮（主色调按钮，图标+文字）。
   - 交互：点击按钮弹出对话框，要求用户输入“小说名称”（必填，1-50字符）。
   - 确认后：调用后端API创建新会话，自动刷新列表并选中新会话，中间区进入创作初始状态。

3. **切换会话**
   - 交互：点击列表中任意非当前选中的会话项。
   - 效果：
     - 左侧列表更新选中状态。
     - 中间区清空当前对话，加载新会话的历史对话记录。
     - 右侧区加载新会话的章节列表。
     - 顶部标题栏显示当前小说名称。

4. **删除会话**
   - 入口：会话列表项的右键菜单（或列表项右侧的“更多”图标按钮）。
   - 交互：
     - 点击“删除”后，弹出**二次确认对话框**。
     - 对话框内容：“删除该会话将同时删除其绑定的所有故事数据（含章节内容、设定、知识库），是否确认删除？”
     - 对话框按钮：
       - “取消”：关闭对话框，不执行操作。
       - “导出后删除”：先触发全本小说导出（见3.4.4），导出完成后自动删除会话。
       - “确认删除”（红色警告按钮）：直接调用后端API删除会话，刷新列表，若删除的是当前会话，自动选中列表中第一个会话（或进入空状态）。

5. **搜索会话**
   - 入口：列表顶部的搜索输入框（带搜索图标，占位符“搜索会话...”）。
   - 交互：实时监听输入框内容变化，根据输入的关键词过滤会话列表（匹配小说名称）。
   - 效果：输入关键词后，列表仅显示名称包含关键词的会话，清空输入后恢复完整列表。

6. **会话状态标识**
   - 标识位置：会话列表项小说名称的左侧。
   - 标识规则：
     - 有未完成创作（最近一次生成未标记完结）：显示蓝色小圆点。
     - 已标记完结：显示灰色“已完结”文字标签。
     - 无特殊状态：不显示标识。

### 3.3 中间对话框区
#### 3.3.1 功能清单
- 对话历史展示（AI输出 + 用户输入）
- AI思考过程与进度可视化
- 用户输入与发送
- 设定/内容确认与修改交互
- 流式输出实时展示
- 快捷命令支持

#### 3.3.2 详细功能描述
1. **对话历史展示**
   - 布局：采用左右分栏的消息流布局。
     - **左侧AI输出区**：展示导演AI、调查AI、作家AI的输出内容，消息气泡左对齐，不同AI角色使用不同的气泡边框色（导演AI：蓝色，调查AI：绿色，作家AI：紫色）。
     - **右侧用户输入区**：展示用户的历史输入内容，消息气泡右对齐，背景色为主题色。
   - 消息内容：
     - AI输出：结构化内容（如设定、大纲）使用折叠面板展示，支持展开/收起；纯文本内容直接展示。
     - 用户输入：原样展示用户输入的文本。
   - 滚动行为：新消息到达时，自动滚动到底部；用户手动向上滚动后，暂停自动滚动，显示“点击滚动到底部”按钮。

2. **AI思考过程与进度可视化**
   - 触发时机：用户发送创作需求后，AI开始处理时显示。
   - 展示位置：中间区顶部，固定显示。
   - 展示内容：
     - 当前阶段文字提示（如“导演AI正在生成设定...”“调查AI正在检索资料...”“作家AI正在创作章节...”）。
     - 进度条：线性进度条，根据当前阶段进度更新（导演AI：0-30%，调查AI：30-60%，作家AI：60-100%）。
     - 思考过程（可选）：若后端返回思考流，实时展示思考内容（使用打字机效果）。
   - 消失时机：当前轮次AI处理完成，最终内容展示后自动消失。

3. **用户输入与发送**
   - 输入区域：
     - 非选择状态：显示单一的多行文本输入框（高度自适应，最大高度200px，超出显示滚动条），右下角“发送”按钮。
     - 选择状态（如AI输出设定后等待确认）：隐藏输入框，显示操作按钮组（“确认”“修改”“取消”）+ 可选的自定义输入框（点击“修改”时显示，用于输入修改意见）。
   - 发送交互：
     - 非选择状态：点击“发送”按钮或按`Ctrl+Enter`发送输入内容。
     - 选择状态：点击“确认”进入下一流程；点击“修改”显示输入框，输入修改意见后点击“发送修改”；点击“取消”返回上一步或结束当前流程。
   - 输入校验：发送前校验输入内容不为空，为空时显示“请输入内容”提示。

4. **设定/内容确认与修改交互**
   - 导演AI设定输出后：
     - 展示结构化设定（使用折叠面板分组展示：故事信息、角色设定、大纲等）。
     - 底部显示“确认进入下一步”“修改设定”按钮。
     - 点击“修改设定”：显示输入框，提示“请告诉我需要修改哪些内容”，用户输入后发送，导演AI重新生成设定。
   - 作家AI章节输出后：
     - 展示章节内容（Markdown格式预览）。
     - 底部显示“确认保存”“修改内容”按钮。
     - 点击“修改内容”：显示输入框，提示“请告诉我需要修改哪些内容（如段落、情节、人物）”，用户输入后发送，作家AI局部修改章节。

5. **流式输出实时展示**
   - 触发：作家AI生成章节内容时，若后端采用流式输出，前端需实时展示。
   - 效果：在中间区AI输出位置，内容逐字（或逐块）显示，模拟打字机效果，同时进度条同步更新。
   - 打断支持：流式输出过程中，输入区域显示“打断”按钮，点击后向后端发送打断请求，停止当前生成，保留已生成内容。

6. **快捷命令支持**
   - 支持命令：在输入框输入`/`显示快捷命令列表。
   - 命令列表：
     - `/exit`：退出当前会话，返回会话选择空状态。
     - `/help`：显示帮助信息（弹出对话框，展示常用操作说明）。
     - `/export`：触发当前小说全本导出。
     - `/finish`：标记当前故事为完结状态。

### 3.4 右侧小说内容区
#### 3.4.1 功能清单
- 章节列表展示
- 章节内容查看与预览
- 章节内容手动修改
- 单章节/全本小说下载
- 章节排序

#### 3.4.2 详细功能描述
1. **章节列表展示**
   - 列表项显示：章节号（如“第一章”）、章节标题（若有）、创建时间（灰色小字）。
   - 排序：
     - 默认：按章节号升序排列（第一章在最上方）。
     - 可选：点击列表顶部的“排序”按钮，切换为“按修改时间倒序”。
   - 选中状态：点击列表项选中，背景色高亮，下方内容区显示对应章节。

2. **章节内容查看与预览**
   - 入口：双击章节列表项，或单击选中后在下方内容区查看。
   - 展示方式：
     - 默认“预览模式”：使用markdown-it渲染Markdown内容，展示格式化后的小说（支持标题、段落、加粗、斜体等基本Markdown语法）。
     - 切换“源码模式”：点击内容区右上角的“源码”按钮，显示原始Markdown文本。
   - 弹窗查看（可选）：双击章节列表项可弹出独立的查看窗口，窗口更大，便于阅读。

3. **章节内容手动修改**
   - 入口：在内容区点击“编辑”按钮（预览模式/源码模式下均可见）。
   - 交互：
     - 进入编辑模式：内容区变为可编辑的Markdown编辑器（推荐使用CodeMirror或Monaco Editor实现，支持语法高亮）。
     - 保存修改：编辑完成后点击“保存”按钮，调用后端API更新章节内容，同时更新Redis与本地文件，显示“保存成功”提示。
     - 取消修改：点击“取消”按钮，放弃未保存的修改，恢复原内容。

4. **单章节/全本小说下载**
   - 单章节下载：
     - 入口：章节列表项右侧的“下载”图标按钮，或章节内容查看窗口的“下载本章”按钮。
     - 效果：调用后端导出接口，下载当前章节的Markdown文件，文件命名规则：`{小说名}_第{章节号}.md`。
   - 全本小说下载：
     - 入口：右侧区顶部的“导出全本”按钮。
     - 效果：调用后端导出接口，下载包含所有章节的ZIP压缩包（或合并为一个Markdown文件），文件命名规则：`{小说名}_全本.zip`。
   - 下载提示：下载开始时显示“正在准备下载...”，下载完成后显示“下载成功”，若失败显示“下载失败，请重试”。

5. **章节排序**
   - 入口：右侧区顶部的“排序”下拉菜单。
   - 选项：
     - “按章节号升序”（默认）。
     - “按修改时间倒序”。
   - 效果：选择后立即重新排序列表。

### 3.5 全局设置与导航
#### 3.5.1 功能清单
- 顶部导航栏
- API Key配置
- 主题配色切换
- 关于/帮助信息

#### 3.5.2 详细功能描述
1. **顶部导航栏**
   - 固定在页面顶部，高度64px。
   - 左侧：系统Logo与名称“WAgent 智能写作”。
   - 中间：当前会话的小说名称（字号16px，加粗），无会话时显示“请选择或创建会话”。
   - 右侧：
     - “设置”按钮（齿轮图标）：点击打开设置抽屉。
     - “帮助”按钮（问号图标）：点击打开帮助对话框。
     - 用户头像/占位图标（可选，预留扩展）。

2. **API Key配置**
   - 入口：设置抽屉的“API配置”标签页。
   - 配置项：
     - 导演AI（豆包）API Key：输入框，默认填充.env中的值，支持修改，修改后实时保存到本地存储。
     - 调查AI（通义千问）API Key：输入框，用户自行配置，保存到本地存储。
     - 作家AI（DeepSeek）API Key：输入框，默认填充.env中的值，支持修改。
   - 验证：输入后点击“验证”按钮，调用后端API测试Key是否有效，显示“验证成功”或“验证失败”提示。

3. **主题配色切换**
   - 入口：设置抽屉的“外观”标签页。
   - 选项：
     - “浅色主题”（默认）：白色背景，黑色文字，蓝色主色调。
     - “深色主题”：深灰色背景，白色文字，蓝色主色调（调整为适合深色的亮度）。
   - 效果：切换后立即应用主题，所有组件颜色同步更新，主题选择保存到本地存储，下次打开自动应用。

4. **关于/帮助信息**
   - 帮助对话框：
     - 内容：系统功能说明、常用操作指南、快捷键列表（如`Ctrl+Enter`发送）。
   - 关于对话框：
     - 内容：系统版本号、更新日期、项目文档链接、技术栈说明。

## 四、UI/UX设计要求
### 4.1 视觉风格规范
- **主色调**：科技蓝（`#409EFF`），用于按钮、链接、选中状态、进度条。
- **辅助色**：
  - 成功：绿色（`#67C23A`），用于调查AI消息、成功提示。
  - 警告：橙色（`#E6A23C`），用于确认对话框。
  - 错误：红色（`#F56C6C`），用于删除按钮、错误提示。
  - 作家AI：紫色（`#909399`调整为`#9B59B6`），用于作家AI消息。
- **字体**：
  - 中文：PingFang SC、Microsoft YaHei、sans-serif。
  - 英文/数字：Roboto、Consolas、monospace。
  - 字号：正文14px，小标题16px，大标题18px-20px。
- **圆角**：组件圆角统一为`4px`或`6px`，避免过于尖锐的直角。
- **阴影**：卡片、对话框使用轻微阴影（`box-shadow: 0 2px 12px 0 rgba(0, 0, 0, 0.1)`），增强层次感。

### 4.2 交互反馈规范
- **加载状态**：
  - 按钮点击后：按钮变为禁用状态，显示加载图标。
  - 页面/数据加载：显示骨架屏或加载动画，避免空白。
- **操作反馈**：
  - 成功操作：右上角弹出成功提示（Element Plus的`ElMessage.success`），3秒后自动消失。
  - 失败操作：弹出错误提示，显示具体错误原因，不自动消失，需用户点击关闭。
  - 确认操作：删除、导出等危险操作必须有二次确认对话框。
- ** hover/点击效果**：
  - 可点击元素（按钮、列表项、图标）：hover时背景色变化（加深/变浅10%），鼠标变为手型。
  - 点击时：添加轻微的缩放效果（`transform: scale(0.98)`），提升点击感。

### 4.3 响应式设计要求
- **桌面端（>1200px）**：标准左中右三栏布局，各区域按比例显示。
- **平板端（768px-1200px）**：
  - 左侧会话区可折叠（点击顶部菜单图标折叠/展开，折叠后宽度60px，仅显示图标）。
  - 右侧小说区可折叠，默认折叠，点击图标展开。
  - 中间区自适应剩余宽度。
- **移动端（<768px）**（可选，最低要求）：
  - 隐藏左右侧栏，仅显示中间对话框区。
  - 底部添加导航栏，切换“会话”“创作”“内容”三个页面。
  - 核心功能（新建会话、发送消息、下载章节）保留，复杂功能（如API配置）可隐藏或简化。

## 五、技术实现要求
### 5.1 项目目录结构
```
web/
├── public/                 # 静态资源
│   ├── favicon.ico         # 网站图标
│   └── logo.png            # 系统Logo

│   │   ├── index.js        # Axios实例配置
│   │   ├── session.js      # 会话相关接口

│   │   └── novel.js        # 小说内容相关接口
```

### 5.2 状态管理（Pinia）
需创建以下Store：
1. **session.js**：
   - 状态：会话列表、当前选中会话ID、加载状态。
   - 动作：获取会话列表、创建会话、删除会话、切换当前会话。

2. **chat.js**：
   - 状态：当前会话的对话历史消息列表、AI处理状态（进行中/空闲）、当前进度。
   - 动作：发送消息、接收AI响应、更新进度、清空对话。

3. **settings.js**：
   - 状态：API Key配置、当前主题、设置抽屉显示状态。
   - 动作：更新API Key、切换主题、保存设置到本地存储。

### 5.3 API通信封装
- 使用Axios创建实例，配置基础URL（开发环境：`http://localhost:8000/api`，生产环境：`/api`）。
- 配置请求拦截器：自动添加Token（若有）、从本地存储读取API Key并添加到请求头。
- 配置响应拦截器：统一处理响应数据，捕获错误（如401未授权、500服务器错误），弹出错误提示。
- 所有API调用返回Promise，使用`async/await`处理。

### 5.4 性能优化要求
- **路由懒加载**：使用Vue Router的动态导入，按需加载页面组件。
- **虚拟列表**：若会话列表或章节列表超过50项，使用虚拟列表（如`vue-virtual-scroller`），减少DOM节点数量。
- **图片懒加载**：图片资源使用懒加载，提升首屏加载速度。
- **本地缓存**：会话列表、设置等不常变的数据缓存到本地存储，减少API请求。
- **防抖/节流**：搜索会话输入框使用防抖（`debounce`，300ms），避免频繁触发过滤。
