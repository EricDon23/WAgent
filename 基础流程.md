WAgent（写作Agent）项目完整流程设计

🎯 项目总览

WAgent是一个三AI协作的自动化故事创作系统，将简单的用户提示转化为完整的高质量故事，支持多轮迭代和用户交互。

🔄 完整工作流程图

用户输入提示词
     ↓
[导演AI] → 生成结构化故事设定
     ↓
用户选择：继续生成 或 修改设定
     ↓
     ├─ 继续生成 ──→ [研究员AI] → 检索资料 → 生成知识库
     │       ↓
     │   [作家AI] → 访问知识库+设定 → 生成故事
     │       ↓
     │   反馈给用户 → 满意？→ 进入下一轮
     │       ↓          否
     │       └─ 用户要求重写 → 重新运行作家AI
     │
     └─ 修改/拓展设定 → 重新运行导演AI

🧩 三个AI模块的独立与协作

1. 导演AI（Director AI）

独立职责：创意结构化、蓝图制定

输入：用户提示词（如"一个晴天"）

输出：完整的
"StorySetting"类（结构化数据）

核心输出格式（LangChain兼容）：

StorySetting:
  - story_name: "故事名称"
  - story_summary: "一句话梗概"
  - story_intro: "简介（200字内）"
  - theme: "核心主旨"
  - characters: Array<CharacterDetail>
  - relationships: "人物关系图"
  - plot_outline: "三幕式大纲"
  - constraints: "创作约束（风格、字数等）"
  - research_needs: "需要研究的主题"

特点：

- 温度=0，确保输出稳定
- 严格遵循JSON Schema
- 可为模糊输入补充合理细节
- 支持迭代修改（基于用户反馈）

2. 研究员AI（Researcher AI）

独立职责：知识收集、背景研究

输入：导演AI输出的
"research_needs"字段

输出：
"KnowledgeBase"类（结构化知识库）

工作流程：

1. 解析
"research_needs"，生成搜索查询
2. 调用工具（MCP/API）检索：
   - 维基百科/百科全书
   - 文学数据库
   - 专业领域资料
   - 作家风格参考
3. 总结归纳为结构化知识
4. 保存到知识库文件系统

输出格式：

KnowledgeBase:
  - research_topic: "研究主题"
  - key_findings: Array<Finding>
  - references: Array<Reference>
  - related_works: "相关作品"
  - style_references: "风格参考"
  - knowledge_graph: "知识图谱（JSON-LD格式）"

工具集成：

- Wikipedia MCP工具
- 学术数据库API
- 文学风格分析器
- 事实核查器

3. 作家AI（Writer AI）

独立职责：故事创作、文学生成

输入：

1. 
"StorySetting"（导演AI输出）
2. 
"KnowledgeBase"（研究员AI输出）
3. 用户可选反馈

输出：

1. 完整故事文本
2. 压缩的
"StoryState"（用于迭代）

创作流程：

访问知识库
    ↓
融合设定 + 知识
    ↓
分轮次生成（每轮一章/节）
    ↓
文学润色 + 连贯性检查
    ↓
输出故事 + 更新状态

StoryState格式：

StoryState:
  - current_progress: "当前进度（章节/字数）"
  - character_developments: "角色发展状态"
  - plot_advancements: "情节推进点"
  - consistency_checks: "连贯性检查点"
  - next_suggestions: "下一步建议"

🔁 完整用户交互流程

阶段1：初始化

用户 → 输入提示词"一个关于AI获得情感的故事"
     ↓
系统 → 调用导演AI
     ↓
导演AI → 生成完整StorySetting
     ↓
前端 → 显示故事设定（名称、梗概、人物等）
     ↓
系统 → 自动提示用户："设定已生成，是否继续创作？"

用户选项：

- A. 继续生成 → 进入阶段2
- B. 修改设定 → 返回导演AI，用户提供修改意见
- C. 拓展设定 → 返回导演AI，添加新元素

阶段2：研究阶段（如果用户选择继续）

系统 → 自动调用研究员AI
     ↓
研究员AI → 分析research_needs
     ↓
     → 调用搜索工具获取资料
     ↓
     → 总结归纳为KnowledgeBase
     ↓
     → 保存到知识库

知识库文件结构：

knowledge_base/
├── topic_1/           # 按主题分类
│   ├── facts.json
│   ├── references.md
│   └── style_guide.md
├── characters/        # 人物研究
│   └── details.json
└── setting/          # 背景设定
    └── worldbuilding.md

阶段3：创作阶段

系统 → 调用作家AI，传入：
     1. StorySetting
     2. KnowledgeBase
     3. 当前轮次信息
     ↓
作家AI → 访问知识库
     ↓
     → 生成第1轮故事（如第1章）
     ↓
     → 保存完整故事
     ↓
     → 生成压缩的StoryState
     ↓
前端 → 显示生成的故事
     ↓
系统 → 提示用户："第1轮完成，是否满意？"

用户选项：

- 满意，继续下一轮 → 作家AI基于StoryState生成第2轮
- 不满意，重新生成 → 重新运行作家AI（可调整参数）
- 修改设定 → 返回导演AI
- 完成 → 结束流程，输出最终故事

🎮 状态管理与迭代机制

状态管理器

class WAgentState:
    """管理整个创作流程的状态"""
    
    def __init__(self):
        self.current_round = 1
        self.story_setting = None
        self.knowledge_base = None
        self.story_state = None
        self.generated_story = []
        self.user_feedback_history = []
        
    def transition(self, user_choice: str, user_input: str = None):
        """状态转移逻辑"""
        if user_choice == "continue":
            if self.current_round == 1:
                # 第一轮：需要先研究
                return "research_phase"
            else:
                # 后续轮次：直接创作
                return "writing_phase"
                
        elif user_choice == "modify_setting":
            return "director_phase"
            
        elif user_choice == "rewrite":
            return "rewrite_phase"

迭代工作流

第1轮：
用户输入 → 导演AI → 研究员AI → 作家AI → 输出第1章

第2轮：
作家AI读取：
  1. 第1轮的StoryState
  2. 原始StorySetting
  3. 第1章内容
  ↓
生成第2章，更新StoryState

StoryState的更新逻辑：

1. 记录新出现的角色发展
2. 标记已解决的情节线索
3. 更新世界观细节
4. 调整后续创作建议

💬 用户反馈处理机制

反馈类型与处理

1. 内容反馈（"主角性格太平面"）→ 更新StorySetting.character_details→ 重新运行导演AI的相应部分
2. 情节反馈（"转折太突然"）→ 更新StoryState.consistency_checks→ 作家AI重新生成时注意铺垫
3. 风格反馈（"文风太正式"）→ 更新StorySetting.constraints.writing_style→ 作家AI调整生成参数
4. 事实反馈（"这个历史细节不对"）→ 更新KnowledgeBase.facts→ 研究员AI重新验证

反馈循环示例

用户：第2章里，主角的反应不太合理
     ↓
系统：分析反馈类型 → 情节逻辑问题
     ↓
     更新StoryState.consistency_checks
     ↓
     提示用户：已记录，下一轮会调整
     ↓
     或：立即重新生成第2章（如果用户要求）

🔧 配置与部署

独立运行模式

每个AI都可以作为独立服务：

# 启动导演AI服务
python director_service.py --port 8001

# 启动研究员AI服务  
python researcher_service.py --port 8002

# 启动作家AI服务
python writer_service.py --port 8003

协作运行模式

通过主协调器连接：

class WAgentCoordinator:
    """主协调器，管理三个AI的协作"""
    
    def process_user_request(self, user_input: str, user_id: str):
        # 1. 调用导演AI
        setting = self.director.generate(user_input)
        
        # 2. 询问用户是否继续
        response = self.ui.ask_continue_or_modify(setting)
        
        if response == "continue":
            # 3. 调用研究员AI
            knowledge = self.researcher.research(setting)
            
            # 4. 调用作家AI
            story = self.writer.write(setting, knowledge)
            
            # 5. 后续迭代
            while not self.ui.is_satisfied(story):
                feedback = self.ui.get_feedback()
                if feedback.type == "rewrite":
                    story = self.writer.rewrite(story, feedback)
