## 一、项目整体架构
### 1.1 五层架构体系（融合基础流程，扩展功能无冗余）
```
交互层（CLI测试+Web前端+本地桌面端）
  ↓
智能体管理层（WAgent核心调度+多智能体生命周期管理）
  ↓
AI处理层（三AI流水线协同，核心业务核心）
  ├─ 输入预处理模块：用户输入标准化+提示词生成
  ├─ 导演AI模块：故事设定生成+状态管理
  ├─ 调查AI模块：资料检索+知识库构建
  └─ 作家AI模块：内容生成+多轮修改
  ↓
工具层（支撑AI层运行，整合所有工具能力）
  ├─ MCP工具服务：百度搜索（Selenium）+ 本地知识库查询（预留接口）
  ├─ 数据处理工具：文本压缩/解压、JSON序列化/反序列化
  ├─ 导出工具：Markdown文件生成与导出
  └─ 自校验工具：设定一致性检查、防吃书检测
  ↓
基础支撑层（保障系统稳定运行）
  ├─ 存储引擎：Redis（高速缓存+持久化）+ 本地文件系统
  ├─ 容器化：Docker（智能体隔离部署）
  ├─ 日志系统：分级日志记录+运行监控
  └─ 异常处理：全局异常捕获+自动重试机制
  ↓
数据层（统一数据管理，分类存储无重复）
  ├─ 会话数据：多会话隔离存储，绑定独立故事与智能体
  ├─ 创作数据：故事设定、章节内容、研究资料、创作进度
  ├─ 智能体数据：智能体状态、运行记录、生命周期信息
  └─ 系统数据：配置参数、日志文件、依赖配置
```

### 1.2 核心业务流程（严格保留原流程，标准化优化）
```
# 核心互动与业务流程（明确会话与故事绑定，每轮对话对应一章创作）
1. 交互入口：打开系统交互界面（Web端/本地桌面端），优先展示已有会话（对话空间）列表，无会话则提示创建新会话。
2. 会话选择：
   - 若存在已有会话（老对话）：用户可选择任一老会话，或创建新会话；
   - 选择老会话：进入对应会话空间，系统自动加载该会话绑定的故事所有数据（全局设定、历史章节、人物关系、知识库等）；
   - 创建新会话：生成全新会话空间，绑定全新空白故事，无任何历史创作数据，后续创作均为全新内容。
3. 单会话多轮对话（每轮对话对应一章内容，一轮对话=导演AI→调查AI→作家AI完整流程）：
   - 第一轮对话（生成第一章）：用户输入初始需求，系统按原有流程执行（输入预处理→导演AI生成全新故事设定/大纲→调查AI构建初始知识库→作家AI生成第一章），自动保存所有数据（故事设定、章节内容、知识库等）并与当前会话绑定。
   - 后续多轮对话（生成后续章节，如第二章、第三章等）：用户在该会话内输入新需求，系统将此需求作为当前章节（如第二章）的生成提示词；
     ① 导演AI：调用该会话绑定的故事全局大纲、历史人物关系，结合用户新需求，新增/修改相关设定，生成当前章节的分支大纲及更新后的人物关系（不偏离原有故事主线）；
     ② 调查AI：基于导演AI更新后的设定，调用已有知识库，补充拓展相关资料，完善知识库并同步保存；
     ③ 作家AI：调用该会话绑定的所有数据（全局设定、当前章节大纲、更新后的人物关系、拓展后的知识库、历史章节内容），生成当前章节完整内容（2000-6000字）；
   - 每轮对话结束后，系统自动保存当前章节内容、更新后的设定及知识库，与当前会话绑定，支持后续多轮对话持续延伸创作。
4. 数据自动保存：所有会话数据（会话信息、绑定的故事设定、历史章节、知识库、创作进度等）均自动同步至Redis+本地文件双重存储，无需用户手动保存，确保数据不丢失。
5. 会话删除逻辑：用户删除老会话时，系统自动弹出提醒窗口，提示“删除该会话将同时删除其绑定的所有故事数据（含章节内容、设定、知识库），是否确认删除？”，并提供“确认删除”“取消”“导出故事后删除”三个选项，避免用户误操作导致数据丢失。
6. 流程闭环：每轮对话完成（生成一章）后，用户可继续在当前会话输入需求，进入下一轮对话（生成下一章），或切换至其他会话、创建新会话，实现多会话、多故事并行创作。
```

## 二、核心模块详细功能要求
本章节整合所有模块要求，无重复内容，严格遵循“不删除、仅扩写优化”原则，每个模块功能明确、权责清晰，贴合技术栈要求。

### 2.1 输入预处理模块
- **输入能力**：支持无约束自由文本输入（如“写一个关于太空探险的故事”）；支持结构化参数输入（故事背景、人物关系、目标字数、风格、文笔、文章类型、章节号）；自动识别混合输入，提取结构化参数并补全缺失值，无需用户额外补充。
- **提示词转换**：基于LangChain的ChatPromptTemplate生成标准化导演AI提示词，固定模板如下：“你是专业小说导演，甲方提出需求：{user_information}。请拓展生成完整的小说创作信息包，包含：1.故事核心主题与整体风格；2.世界观背景与时代设定；3.主要角色设定（姓名、性格、外貌、动机）；4.核心人物关系图谱；5.全本故事总体大纲与走向（仅第一章生成）；6.当前章节详细分镜大纲与情节走向。要求：逻辑严谨、无矛盾，温度=0，输出结构化JSON格式”。
- **参数补全**：默认字数3000字/章，默认风格为现实主义，默认文笔为流畅自然，默认文章类型为长篇小说，默认章节号为1（新故事场景），补全参数不影响用户自定义输入。

### 2.2 导演AI模块
- **故事状态判断**：检索Redis中`story:{story_id}:meta`键值对，自动识别当前任务为“全新故事”或“已有故事续写”，无冗余判断逻辑，确保响应高效。
- **差异化输出规则**：全新故事需输出全本总体大纲、故事走向、第一章详细大纲及完整基础设定包（含世界观、角色、人物关系）；续写故事仅输出当前章节详细分镜大纲，严格遵循已存储的全局设定与故事主线，不偏离原有设定。
- **状态管理**：内置基础记忆功能，自动记录每个故事的“完结状态”“当前章节数”“最后更新时间”，所有状态数据存储于Redis `story:{story_id}:progress` 键，实时更新，可随时调用查询。
- **交互逻辑**：输出设定后，提供“确认进入下一步”“修改设定”两个选项；后台测试通过输入y/n控制流程，前端通过对话框按钮触发操作；支持多轮修改，每次修改后重新生成设定并同步更新Redis中的对应数据，确保数据一致性。
- **模型参数**：强制温度=0，确保输出内容稳定、逻辑严谨，无随机偏差，符合结构化输出要求；输出格式严格遵循Pydantic `StorySetting` 数据模型，使用`PydanticOutputParser` 解析，避免格式混乱。

### 2.3 导演-研究者传递模块
- **数据持久化**：接收导演AI输出的结构化设定，自动拆解为6类数据，分类存入Redis，对应Key前缀规范如下，无重复存储：
  - `story:{story_id}:meta`：小说基础信息（标题、类型、创建时间）
  - `story:{story_id}:global_outline`：全本总体大纲与故事走向
  - `story:{story_id}:chapter:{chapter_id}:outline`：单章节大纲
  - `story:{story_id}:characters`：角色设定库
  - `story:{story_id}:relations`：人物关系图谱
  - `story:{story_id}:progress`：故事进度与状态
- **提示词生成**：基于导演AI输出的设定内容，生成调查AI专属提示词，固定模板如下：“你是专业文学研究者，根据以下小说创作信息，检索并整理所需的权威资料：{director_information}。检索范围：历史背景、行业常识、地理环境、文化习俗、文学风格参考、情节逻辑依据。要求：信息准确、来源可靠，温度=0，输出结构化资料包”，确保提示词精准，贴合调查需求。

### 2.4 调查AI模块
- **工具调用**：基于MCP框架调用`baidu_search`工具，通过Selenium实现浏览器自动化联网检索；预留本地知识库查询接口，支持后续扩展，不影响当前核心功能。
- **资料处理**：对检索到的网络资料进行去重、筛选、总结，提取与当前小说创作相关的核心信息，剔除无关内容；自动生成结构化知识库，包含研究主题、关键发现、参考文献，确保资料可直接被作家AI调用。
- **数据入库**：将整理后的结构化资料存入Redis，Key前缀为`story:{story_id}:research:{chapter_id}`，与对应故事、章节绑定，便于精准调用，避免资料混乱。
- **模型参数**：强制温度=0，保证资料的客观性、准确性，无主观偏差，贴合文学研究的严谨性要求。

### 2.5 作家AI输入整合模块
- **数据拉取**：从Redis中精准拉取对应故事的全局设定、当前章节大纲、调查研究资料，同时自动加载用户指定的字数、风格、文笔约束，无需手动输入，确保数据完整。
- **提示词拼接**：生成标准化作家AI提示词，模板如下：“你是专业小说作家，基于以下所有信息生成符合要求的小说章节：【全局设定】{global_settings}【当前章节大纲】{chapter_outline}【参考资料】{research_materials}。要求：1.字数严格控制在{min_word}-{max_word}字；2.风格：{style}，文笔：{writing_style}；3.严格遵循所有设定，不得出现吃书内容；4.情节流畅、逻辑自洽、人物行为符合性格”，确保提示词包含所有核心约束。

### 2.6 作家AI模块
- **内容生成**：基于整合后的提示词，生成对应章节的小说正文，严格执行2000-6000字的单章字数要求，支持用户自定义调整字数范围，确保内容符合要求。
- **多轮修改支持**：接收用户的具体修改指令，针对指定段落、情节、人物进行局部修改，无需重新生成全文，提升修改效率，减少冗余操作。
- **中途打断支持**：允许用户在内容生成过程中随时打断，输入新的指令调整故事走向、风格或情节，系统实时响应，不丢失已生成内容。
- **记忆调用**：生成内容前，自动读取Redis中存储的全局设定与历史章节内容，确保人物性格、世界观设定、情节发展的一致性，杜绝吃书问题。

### 2.7 总结与存储模块
- **内容存储**：将作家AI生成的小说正文以Markdown格式存入Redis，Key前缀为`story:{story_id}:chapter:{chapter_id}:content`；同时在本地文件系统保存备份，路径为`stories/{story_id}/drafts/chapter_{chapter_id}.md`，双重存储确保数据安全；会话与故事一一绑定，存储数据均关联会话ID与故事ID，便于精准调用与管理。
- **设定更新**：自动从新生成的章节内容中提取新增的设定、人物、情节，同步更新到Redis对应的全局设定库中，确保后续章节创作不偏离原有设定，从源头防止吃书；更新后的数据同步绑定至当前会话，供下一轮对话（下一章创作）调用。
- **进度更新**：自动更新故事的“当前章节数”“最后更新时间”，当用户指定故事完结时，同步标记故事完结状态，更新至`story:{story_id}:progress`；进度数据与会话绑定，切换会话后可实时查看对应故事的创作进度。
- **导出支持**：支持将单章节或全本小说导出为Markdown格式文件，导出功能可独立调用，也可在删除会话时触发（提供“导出故事后删除”选项），导出文件包含完整的章节结构、设定及知识库摘要，可直接用于编辑、备份或分享。

### 2.8 WAgent智能体整合模块
- **统一调度**：作为系统中枢，统一调度所有模块的执行顺序与数据流转，基于LangGraph实现状态机式流程控制，确保流程顺畅，无模块脱节。
- **多线程优化**：采用多线程技术，并行执行资料检索、数据处理等非阻塞任务，提升系统整体运行速度，缩短章节生成与操作响应时间。
- **标准化接口**：基于LangChain Agent框架，实现模块间的标准化通信与接口调用，确保模块间兼容性，便于后续扩展与维护。
- **错误处理**：内置全局异常捕获与重试机制，自动处理网络异常、API调用失败、工具调用超时等常见问题，避免系统崩溃，提升稳定性。
- **日志集成**：自动记录所有模块的运行日志，与系统日志模块联动，支持日志分级与检索，便于问题排查。

### 2.9 数据存储与调用层
#### 2.9.1 存储模块
- **存储引擎**：基于Redis实现全量数据存储，所有数据采用JSON格式序列化存储，便于解析与调用；本地文件系统用于持久化小说内容与日志文件，双重保障数据不丢失。
- **数据优化**：对大文本数据（如小说章节、研究资料）进行gzip压缩处理，减少Redis内存占用；采用分层存储结构，热点数据（常用会话、当前创作故事）常驻内存，冷数据（历史会话、旧章节）定期持久化到磁盘，优化存储性能。
- **混合存储**：支持本地化存储与云端存储的统一管理，数据自动同步，确保多端访问数据一致，兼顾离线使用与云端备份需求。
- **数据分类**：严格按照“元数据、设定数据、内容数据、研究数据、日志数据”分类存储，每个故事分配唯一ID，所有数据与故事ID绑定，避免数据混乱与重复。

#### 2.9.2 调出模块
- **数据解析**：提供统一的数据读取接口，自动解析Redis中存储的压缩JSON数据，无需用户手动解析，确保数据调用便捷。
- **精准检索**：支持按故事ID、章节ID、关键词等维度快速检索数据，检索响应时间≤100ms，提升数据调用效率。
- **批量导出**：支持批量导出指定故事的所有相关数据，导出格式支持JSON（用于数据迁移）与Markdown（用于内容编辑），满足不同使用需求。

### 2.10 会话与智能体管理模块
- **容器化隔离**：基于Docker实现每个WAgent智能体的独立容器化部署，每个智能体拥有独立的运行环境，保证多智能体之间互不干扰，提升系统稳定性。
- **会话管理（核心优化）**：基于Redis的RedisChatMessageHistory实现多会话（对话空间）隔离，每个会话独立绑定一个故事，会话与故事一一对应，数据互不干扰；支持五大核心操作：
  - 开启新对话（新会话）：生成全新会话空间，绑定全新空白故事，无任何历史创作数据，后续创作均为全新内容；
  - 调用历史对话（老会话）：选择已有会话后，自动加载该会话绑定的所有故事数据（全局设定、历史章节、人物关系、知识库等），支持基于历史内容延伸创作；
  - 切换不同对话：在多个会话间自由切换，切换后自动加载对应会话的故事数据，不影响各会话的创作进度；
  - 按关键词搜索历史对话：通过关键词快速检索目标会话，提升会话查找效率；
  - 删除不需要的对话：删除会话时，系统自动弹出提醒，提示“删除该会话将同时删除其绑定的所有故事数据（含章节内容、设定、知识库），是否确认删除？”，提供“确认删除”“取消”“导出故事后删除”三个选项，规避误操作风险。
- **智能体生命周期管理**：支持智能体全生命周期操作，包括开启新智能体、关闭运行中的智能体、切换不同智能体、重启历史智能体、删除已有的智能体，操作便捷，可灵活管理多个智能体，智能体与会话联动，确保创作流程顺畅。
- **入口逻辑**：系统启动后，自动进入智能体选择界面；用户选择智能体后，进入会话列表界面（展示所有已有会话），可选择老会话、创建新会话；进入对应会话后，即可输入需求开展创作；退出对话后，返回会话列表界面，流程清晰，符合用户使用习惯。
- **会话与故事绑定规则**：会话为故事的载体，一个会话仅绑定一个故事，一个故事仅对应一个会话；会话的所有操作（修改、删除、切换）均同步关联其绑定的故事数据，确保数据一致性；删除会话则同步删除其绑定的所有故事数据（需经用户确认并提供导出选项）。

### 2.11 系统运维与监控模块
- **日志模块**：记录系统所有模块的运行日志，包括用户操作日志、AI调用日志、错误异常日志、数据读写日志；日志按级别分级存储（DEBUG、INFO、WARNING、ERROR），支持按时间、模块、级别检索日志，便于系统运维与问题排查。
- **后台启动模块**：提供一键启动脚本`WAgenter.bat`，运行脚本后自动启动所有依赖服务与核心模块，无需手动逐一启动；提供可视化启动界面，实时显示各模块运行状态，便于用户查看系统启动情况。
- **WAgenter.bat控制塔模式（核心新增）**：脚本内置控制台控制塔模式，无需Web前端即可实现系统核心功能操作，支持纯命令行交互，适合开发测试与无图形界面环境使用。控制塔模式功能如下：
  - 启动入口：运行`WAgenter.bat`后，系统自动启动Redis服务、MCP工具服务及核心AI服务，成功后进入控制塔主界面。
  - 主菜单功能：显示数字选项菜单，支持输入数字选择操作：
    1. 选择已有会话：列出所有存在的会话（显示会话ID、绑定的小说名称、最后更新时间），用户输入会话编号即可选择对应会话。
    2. 新增会话：提示用户输入小说名称，系统自动创建全新会话并绑定空白故事，生成唯一会话ID。
    3. 删除会话：列出所有存在的会话，用户输入会话编号后，系统弹出确认提示："删除该会话将同时删除其绑定的所有故事数据，是否确认？[Y/N/D(导出后删除)]"，支持直接删除、取消或导出故事后删除。
    4. 查看故事文件夹位置：列出所有存在的会话，用户输入会话编号后，系统显示该会话绑定的故事在本地文件系统的完整路径，并自动打开对应文件夹。
    5. 退出系统：安全关闭所有服务，退出控制塔模式。
  - 会话对话模式：选择会话后进入Agent对话界面，用户可输入创作需求，系统按完整流程执行（导演AI→调查AI→作家AI），实时显示各阶段进度与输出内容；对话过程中输入`/exit`可退出对话，返回控制塔主界面；输入`/help`可查看可用命令。
  - 状态显示：控制塔界面实时显示系统运行状态、当前会话信息、已生成章节数等关键信息，便于用户掌握系统运行情况。
- **server-Wagent.bat脚本（新增）**：专注于一键启动后端服务器与Web前端，无需手动分别启动服务，简化Web端访问流程，适合日常创作使用，核心功能如下：
  - 启动入口：双击运行`server-Wagent.bat`，系统自动检查虚拟环境、依赖服务及核心配置，无需额外操作。
  - 自动启动服务：依次启动Redis服务（若未启动）、FastAPI后端服务器（基于Uvicorn），确保后端服务正常运行。
  - 自动打开前端：后端服务器启动成功后，自动调用系统默认浏览器，打开Web前端访问地址（http://localhost:8000），直接进入会话列表界面，可立即开展创作。
  - 状态提示：脚本运行过程中实时显示各步骤状态（如“正在启动Redis服务”“后端服务器启动成功”“正在打开Web前端”），启动失败时给出明确错误提示（如端口占用、依赖缺失），便于排查问题。
  - 退出逻辑：关闭脚本窗口即可停止后端服务器与Redis服务，确保服务安全退出，不残留进程。
- **测试模块**：内置模拟数据测试套件，支持对所有模块进行单独测试与集成测试；测试完成后自动生成测试报告，明确测试结果与问题，确保所有模块功能正常。

### 6.2 部署步骤（清晰可执行，无冗余步骤）
1. 创建虚拟环境：使用conda创建独立虚拟环境，避免依赖冲突，执行命令：`conda create -n wagent python=3.10`
2. 激活虚拟环境：激活已创建的虚拟环境，执行命令：`conda activate wagent`
3. 安装依赖：执行命令 `pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple`，安装所有依赖包，确保版本与文档要求一致；注意：若出现“网页访问失败，请稍后重试”报错，需检查网络连接，或替换为其他可用的PyPI镜像源（如阿里云镜像源）后重新执行。
4. 启动Redis：使用Docker启动Redis服务，执行命令：`docker run -d --name redis-wagent -p 6379:6379 redis:latest`，启动后确认Redis服务正常运行。
5. 配置.env文件：复制.env模板，填入个人阿里云通义千问API Key，其他参数保持默认，无需修改。
6. 启动系统：提供两种启动方式，按需选择：
   - 方式1：运行`WAgenter.bat`，启动所有依赖服务与控制塔模式，支持命令行交互与Web端访问。
   - 方式2：运行`server-Wagent.bat`，一键启动后端服务器与Redis服务，自动打开Web前端，专注于Web端可视化创作。
7. 访问使用：Web端访问`http://localhost:8000`，或直接运行本地桌面端`WAgent.exe`，即可开展创作。
