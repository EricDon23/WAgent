## 三、完整工作流程（严格遵循《基础流程.md》）
### 阶段1：导演AI（Director AI）
**技术实现**：LangChain `ChatPromptTemplate` + `PydanticOutputParser` + LCEL链式调用
- **输入**：用户提示词（如"一个关于AI获得情感的科幻故事"）
- **输出**：严格符合JSON Schema的`StorySetting`对象
- **核心约束**：
  1. 温度=0，确保输出100%稳定、无随机内容
  2. 所有字段必须完整，缺失则自动补全合理值
  3. 自动生成`research_needs`字段，供研究员AI使用
- **Pydantic数据模型**（强制校验）：
  ```python
  class CharacterDetail(BaseModel):
      name: str = Field(description="角色名称")
      role: str = Field(description="角色身份")
      personality: str = Field(description="角色性格")
      background: str = Field(description="角色背景")

  class StorySetting(BaseModel):
      story_name: str = Field(description="故事名称")
      story_summary: str = Field(description="一句话梗概")
      story_intro: str = Field(description="200字内简介")
      theme: str = Field(description="核心主旨")
      characters: list[CharacterDetail] = Field(description="主要角色列表")
      relationships: str = Field(description="人物关系图")
      plot_outline: str = Field(description="三幕式大纲")
      constraints: str = Field(description="创作约束（风格、字数等）")
      research_needs: list[str] = Field(description="需要研究的主题列表")
  ```

### 阶段2：研究员AI（Researcher AI）
**技术实现**：MCP协议（stdio通信）+ FastMCP + Selenium百度搜索
- **输入**：导演AI输出的`research_needs`字段
- **输出**：结构化`KnowledgeBase`对象
- **核心流程**：
  1. 解析研究需求，生成搜索关键词
  2. 调用MCP百度搜索工具，获取前5条搜索结果
  3. 总结归纳为结构化知识
  4. 保存为`knowledge_base.json`，供作家AI直接调用
- **MCP百度搜索服务**（`baidu_search_server.py`）：
  - 基于FastMCP构建，使用stdio通信
  - 无头模式运行Chrome，自动处理页面加载
  - 返回标题、摘要、链接，格式统一为字符串

### 阶段3：作家AI（Writer AI）
**技术实现**：LangChain `RunnableWithMessageHistory` + Redis记忆
- **输入**：
  1. `StorySetting`（导演AI）
  2. `KnowledgeBase`（研究员AI）
  3. `StoryState`（G模块记忆）
- **输出**：
  1. 完整故事文本（保存为`round_X.md`）
  2. 更新后的`StoryState`（存入Redis）
- **核心约束**：
  1. 生成前必须读取G模块记忆，无记忆则新建
  2. 严格遵循`StorySetting`中的创作约束
  3. 自动检查章节属性，非初始章节必须调用前一章内容

### 阶段4：G模块（数据记忆模块）
**技术实现**：Redis + `RedisChatMessageHistory`
- **核心功能**：约束后续生成，防止内容偏移
- **触发逻辑**：每次生成前自动检测记忆
  - 有记忆 → 按记忆生成
  - 无记忆 → 新建记忆并存储
- **存储内容**：
  - 故事设定（StorySetting）
  - 已生成章节内容
  - 角色发展状态
  - 情节推进点
- **持久化**：Redis TTL=72小时，程序重启后记忆不丢失
