## 一、桌面端概述
### 1.1 定位与目标
本地桌面端是WAgent系统的核心交互界面之一，与Web前端功能**100%对齐**，为用户提供离线、高性能、视觉炫酷的本地小说创作体验。桌面端需充分利用Windows平台特性，实现系统托盘集成、窗口状态记忆、离线数据缓存等特色功能，同时确保与Web端数据实时同步（在线时）。

### 1.2 核心设计原则
- **功能一致性**：与Web前端功能完全对齐，会话、创作数据多端实时同步（在线时）。
- **离线优先**：支持完整离线创作，数据本地优先存储，在线后自动同步云端。
- **视觉炫酷**：采用现代化WPF界面，支持流畅动画、深色/浅色主题切换、毛玻璃效果等。
- **性能优化**：启动时间<3秒，章节生成响应时间与Web端一致，无明显卡顿。
- **平台集成**：深度集成Windows平台特性（系统托盘、跳转列表、通知中心）。

## 三、功能需求详细说明
### 3.1 整体窗口结构（左中右三区布局）
主窗口采用固定的左中右三栏布局，各区域功能与Web端完全一致，支持窗口大小调整，各区域宽度可按比例拖动调整。

| 区域 | 初始宽度占比 | 最小宽度 | 可调整 | 核心功能 |
|------|--------------|----------|--------|----------|
| 左侧会话选择区 | 22% | 260px | 是 | 会话列表展示、新建/切换/删除/搜索会话 |
| 中间对话框区 | 48% | 580px | 是 | AI输出展示、用户输入、创作流程交互 |
| 右侧小说内容区 | 30% | 300px | 是 | 章节列表展示、章节内容查看/修改/下载 |

#### 窗口特性
- **窗口状态记忆**：记住用户上次关闭时的窗口位置、大小、各区域宽度比例，下次启动自动恢复。
- **最小化到托盘**：点击最小化按钮时，窗口隐藏到系统托盘，双击托盘图标重新显示。
- **全屏模式**：支持F11切换全屏/窗口模式，全屏时隐藏标题栏与任务栏。
- **跳转列表**：Windows任务栏右键跳转列表显示“最近打开的3个会话”，点击快速打开对应会话。

### 3.2 左侧会话选择区
#### 3.2.1 功能清单
- 会话列表展示（带缩略预览）
- 新建会话（支持模板选择）
- 切换会话
- 删除会话（带回收站恢复）
- 搜索会话（支持模糊匹配）
- 会话状态标识与进度条
- 会话右键菜单

#### 3.2.2 详细功能描述
1. **会话列表展示**
   - 列表项设计：采用卡片式布局，每个会话项显示：
     - 左侧：小说封面占位图（或自动生成的首字母头像）。
     - 右侧上半部分：小说名称（主标题，16px，加粗）、最后更新时间（12px，灰色）。
     - 右侧下半部分：简短进度条（显示已完成章节数/计划章节数）、状态标签（“未完结”/“已完结”）。
   - 排序与分组：
     - 默认按最后更新时间倒序排列。
     - 支持分组：“最近打开”“未完结”“已完结”。
   - 选中状态：当前选中的会话项使用主题色高亮，添加轻微阴影效果。
   - 空状态：无会话时，显示“开始你的创作之旅”插画+“新建会话”按钮。

2. **新建会话（支持模板选择）**
   - 入口：列表顶部的“+ 新建会话”大按钮（主色调，圆角设计）。
   - 交互：点击弹出**新建会话对话框**（模态窗口），包含：
     - 第一步：选择模板（“空白模板”“科幻模板”“言情模板”“悬疑模板”，模板内置基础设定）。
     - 第二步：输入小说名称（必填，1-50字符）、选择类型（下拉框：长篇小说/短篇小说/散文）。
     - 第三步：确认创建，显示加载动画。
   - 确认后：自动刷新列表并选中新会话，中间区进入创作初始状态，同时在任务栏跳转列表添加该会话。

3. **切换会话**
   - 交互：点击列表中任意非当前选中的会话项。
   - 效果：
     - 左侧列表更新选中状态，添加平滑过渡动画。
     - 中间区清空当前对话，加载新会话的历史对话记录（支持增量加载，历史消息过多时显示“加载更多”）。
     - 右侧区加载新会话的章节列表，使用骨架屏过渡。
     - 窗口标题栏更新为“WAgent - {小说名称}”。

4. **删除会话（带回收站恢复）**
   - 入口：会话列表项的右键菜单 → “删除”，或列表项右侧的“垃圾桶”图标。
   - 交互：
     - 点击后弹出**二次确认对话框**：“删除该会话将同时删除其绑定的所有故事数据，是否确认删除？”
     - 对话框按钮：
       - “取消”：关闭对话框。
       - “导出后删除”：先触发全本小说导出，完成后删除。
       - “移到回收站”（默认）：将会话移到本地回收站，不真正删除。
   - 回收站功能：
     - 入口：左侧列表底部的“回收站”按钮。
     - 界面：显示所有已删除的会话，支持“恢复”或“永久删除”。
     - 保留策略：回收站中的会话保留30天，30天后自动永久删除。

5. **搜索会话**
   - 入口：列表顶部的搜索框（带放大镜图标，占位符“搜索小说名称、角色或内容...”）。
   - 交互：实时监听输入，支持模糊匹配（小说名称、角色名、章节内容关键词）。
   - 效果：输入后列表实时过滤，高亮匹配的关键词，显示“找到X个结果”。

6. **会话状态标识与进度条**
   - 状态标识：
     - 未完结：蓝色小圆点+“未完结”标签。
     - 已完结：灰色对勾+“已完结”标签。
     - 有未保存修改：橙色感叹号+“未保存”标签。
   - 进度条：会话项底部的细进度条，显示已完成章节数/计划章节数（如“3/10”）。

7. **会话右键菜单**
   - 右键点击会话项显示菜单：
     - “打开”：切换到该会话。
     - “重命名”：修改小说名称。
     - “导出全本”：导出该会话的所有章节。
     - “移到回收站”：删除会话。
     - “打开文件夹”：打开该会话在本地文件系统的存储目录。

### 3.3 中间对话框区
#### 3.3.1 功能清单
- 对话历史展示（支持时间线视图）
- AI思考过程与3D进度可视化
- 用户输入（支持富文本编辑）
- 设定/内容确认与修改交互
- 流式输出实时展示（打字机效果）
- 快捷命令与快捷键支持
- 中途打断与暂停/继续

#### 3.3.2 详细功能描述
1. **对话历史展示（支持时间线视图）**
   - 布局：采用**时间线视图**（替代Web端的左右分栏），消息按时间顺序从上到下排列，左侧时间线显示AI角色图标（导演/调查/作家）。
   - 消息气泡设计：
     - AI消息：左对齐，不同AI角色使用不同主题色边框（导演：蓝色，调查：绿色，作家：紫色），支持展开/折叠结构化内容（如设定、大纲）。
     - 用户消息：右对齐，背景色为主题色，显示用户头像/首字母。
   - 消息操作：
     - 右键菜单：“复制”“重新生成”（仅AI消息）“删除”。
     - 悬停效果：显示消息时间戳。
   - 滚动与加载：
     - 新消息自动滚动到底部，带平滑滚动动画。
     - 历史消息超过50条时，顶部显示“加载更早消息”按钮，支持分页加载。

2. **AI思考过程与3D进度可视化**
   - 触发时机：用户发送创作需求后显示。
   - 展示位置：中间区顶部的**固定状态栏**。
   - 展示内容：
     - 左侧：3D旋转的AI角色图标（WPF 3D Viewport实现，或使用Lottie动画）。
     - 中间：当前阶段文字（如“导演AI正在构建世界观...”）+ 动态进度条（带粒子效果）。
     - 右侧：思考过程面板（可折叠），实时显示AI的思考流（打字机效果）。
   - 消失时机：当前轮次完成后，状态栏平滑收起，显示“本轮完成”提示。

3. **用户输入（支持富文本编辑）**
   - 输入区域设计：
     - 非选择状态：**富文本编辑器**（支持加粗、斜体、列表、插入角色名快捷按钮），高度自适应（最大300px），右下角“发送”按钮（支持`Ctrl+Enter`）。
     - 选择状态：隐藏编辑器，显示操作按钮组（“确认”“修改”“取消”）+ 可选的修改意见输入框。
   - 快捷按钮：
     - 编辑器顶部工具栏：“插入角色”（下拉选择已定义角色）“插入设定”“清空”。
   - 输入校验：发送前校验内容不为空，为空时显示抖动动画+提示。

4. **设定/内容确认与修改交互**
   - 导演AI设定输出后：
     - 使用**折叠面板**分组展示设定（“故事信息”“角色设定”“世界观”“大纲”），每个面板可独立展开/收起。
     - 底部悬浮操作栏：“确认进入下一步”“修改设定”。
     - 点击“修改设定”：显示**修改对话框**，支持直接编辑JSON或使用表单修改，修改后点击“重新生成”。
   - 作家AI章节输出后：
     - 显示Markdown预览，底部悬浮栏：“确认保存”“修改内容”“重新生成”。
     - 点击“修改内容”：切换到编辑模式，用户可直接修改Markdown，或输入修改指令让AI重写。

5. **流式输出实时展示（打字机效果）**
   - 触发：作家AI生成章节时。
   - 效果：
     - 消息气泡逐字显示内容，模拟打字机效果，光标闪烁。
     - 进度条实时更新，显示已生成字数/目标字数。
     - 支持**语法高亮**：Markdown内容实时渲染，标题、加粗等格式即时生效。

6. **快捷命令与快捷键支持**
   - 快捷命令：输入`/`显示命令列表（与Web端一致）：
     - `/exit`：返回会话列表。
     - `/help`：打开快捷键帮助窗口。
     - `/export`：导出全本。
     - `/finish`：标记完结。
   - 全局快捷键：
     - `Ctrl+N`：新建会话。
     - `Ctrl+S`：保存当前内容。
     - `Ctrl+E`：导出全本。
     - `F11`：全屏。
     - `Esc`：取消/关闭对话框。

7. **中途打断与暂停/继续**
   - 打断：流式输出过程中，输入区域显示“打断”按钮，点击后停止生成，保留已生成内容。
   - 暂停/继续：生成过程中支持暂停（显示“暂停”按钮），暂停后可继续。

### 3.4 右侧小说内容区
#### 3.4.1 功能清单
- 章节列表展示（带缩略预览）
- 章节内容查看（双视图：预览/源码）
- 章节内容手动修改（支持语法高亮）
- 单章节/全本小说下载与分享
- 章节排序与拖拽调整
- 章节评论与批注（可选，扩展功能）

#### 3.4.2 详细功能描述
1. **章节列表展示（带缩略预览）**
   - 列表项设计：
     - 显示章节号、章节标题、创建时间、字数统计。
     - 悬停时显示章节首句预览（Tooltip）。
   - 拖拽调整：支持拖拽列表项调整章节顺序，调整后自动更新章节号。
   - 排序：
     - 默认按章节号升序。
     - 可选按修改时间、字数排序。
   - 选中状态：高亮显示，右侧内容区加载对应章节。

2. **章节内容查看（双视图：预览/源码）**
   - 双视图切换：
     - **预览模式**：使用Markdig.Wpf渲染，支持目录导航（点击标题跳转）、图片显示、代码高亮。
     - **源码模式**：使用AvalonEdit编辑器，支持Markdown语法高亮、行号显示、自动补全。
   - 弹窗查看：双击章节列表项弹出独立的**查看窗口**，支持窗口最大化，便于沉浸式阅读。

3. **章节内容手动修改（支持语法高亮）**
   - 入口：内容区顶部的“编辑”按钮。
   - 交互：
     - 切换到源码编辑模式，AvalonEdit编辑器获得焦点。
     - 支持**实时预览**：编辑器右侧显示同步预览（分栏视图）。
     - 保存：点击“保存”按钮（或`Ctrl+S`），显示保存成功动画，同步更新本地与云端数据。
     - 撤销/重做：支持无限撤销/重做。

4. **单章节/全本小说下载与分享**
   - 单章节下载：
     - 入口：章节列表项右键 → “下载本章”，或查看窗口的“下载”按钮。
     - 格式：支持Markdown（.md）、PDF（可选，需要PDF渲染库）、TXT。
     - 保存对话框：使用Windows原生保存文件对话框，默认文件名：`{小说名}_第{章节号}.md`。
   - 全本小说下载：
     - 入口：右侧区顶部的“导出全本”大按钮。
     - 格式：ZIP压缩包（包含所有章节.md+设定.json）、合并为单个Markdown、EPUB（可选）。
   - 分享：
     - 支持“复制到剪贴板”“分享到微信”（调用系统分享）。

5. **章节排序与拖拽调整**
   - 拖拽：在列表中长按并拖拽章节项，调整顺序，释放后自动重新编号。
   - 排序按钮：顶部下拉菜单选择排序方式。

### 3.5 全局设置与导航
#### 3.5.1 功能清单
- 顶部导航栏与汉堡菜单
- API Key配置与模型选择
- 主题配色切换（支持自定义主题）
- 系统托盘与通知
- 关于/帮助与检查更新

#### 3.5.2 详细功能描述
1. **顶部导航栏与汉堡菜单**
   - 顶部导航栏：
     - 左侧：汉堡菜单按钮（展开/收起左侧会话区）、系统Logo、当前小说名称。
     - 中间：搜索框（全局搜索）。
     - 右侧：“设置”按钮、“帮助”按钮、窗口控制按钮（最小化/最大化/关闭）。
   - 汉堡菜单：
     - 点击展开侧边栏，显示：“会话列表”“回收站”“设置”“帮助”“关于”“退出”。

2. **API Key配置与模型选择**
   - 入口：设置窗口 → “模型配置”标签页。
   - 配置项：
     - 导演AI：模型选择（下拉框：DouBao-seed-2.0-pro）、API Key输入框、“验证”按钮。
     - 调查AI：模型选择（通义千问）、API Key输入框。
     - 作家AI：模型选择（DeepSeek）、API Key输入框。
   - 保存：配置自动保存到SQLite数据库，下次启动自动加载。

3. **主题配色切换（支持自定义主题）**
   - 入口：设置窗口 → “外观”标签页。
   - 预设主题：
     - “浅色主题”（默认）：白色背景，蓝色主色调。
     - “深色主题”：深灰背景，紫色主色调，毛玻璃效果。
     - “护眼模式”：豆绿色背景，暖色调。
   - 自定义主题：
     - 支持自定义主色调、背景色、字体，实时预览，保存为自定义主题。
   - 切换效果：主题切换时使用平滑过渡动画（1秒），所有控件颜色同步更新。

4. **系统托盘与通知**
   - 系统托盘：
     - 最小化时隐藏到托盘，托盘图标显示未读消息数（红色角标）。
     - 右键菜单：“打开WAgent”“最近会话”“设置”“退出”。
   - 通知：
     - 章节生成完成、导出完成等事件显示Windows 10/11原生通知（Toast Notification）。
     - 通知可点击，点击后打开对应会话。

5. **关于/帮助与检查更新**

```
WAgent.Desktop/
├── WAgent.Desktop.sln       # 解决方案文件
├── src/
│   ├── WAgent.App/          # 主应用程序项目
│   │   ├── App.xaml          # 应用程序入口
│   │   ├── App.xaml.cs
│   │   ├── MainWindow.xaml   # 主窗口
│   │   └── Assets/           # 资源文件（图标、样式）
│   ├── WAgent.Core/         # 核心逻辑项目
│   │   ├── Models/           # 数据模型（Session、Chat、Novel等）
│   │   ├── ViewModels/       # 视图模型（MainViewModel、SessionViewModel等）
│   │   ├── Services/         # 服务（ApiService、StorageService、NotificationService等）
│   │   ├── Helpers/          # 辅助类（JsonHelper、DependencyInjectionHelper等）
│   │   └── Constants/        # 常量（ApiUrls、ThemeKeys等）

├── tests/                    # 单元测试项目
└── setup/                    # 安装包项目（Inno Setup脚本）
```

### 5.2 核心服务实现
1. **ApiService**：
   - 封装HttpClient，实现所有后端API调用（会话、对话、小说、导出）。
   - 实现重试机制（网络失败时自动重试3次）。
   - 实现离线缓存：无网络时返回本地缓存数据。

2. **StorageService**：
   - 使用SQLite存储会话、设置、离线缓存。
   - 实现异步CRUD操作，避免阻塞UI线程。
   - 使用`Microsoft.Data.Sqlite`进行数据库操作。

3. **NotificationService**：
   - 封装系统托盘与通知功能。
   - 使用`Hardcodet.NotifyIcon.Wpf`实现托盘图标。
   - 使用Windows Community Toolkit的`ToastNotification`实现原生通知。

4. **ThemeService**：
   - 管理主题切换，动态加载资源字典。
   - 保存用户选择的主题到本地设置。

### 5.3 状态管理与MVVM
- 使用`CommunityToolkit.Mvvm`的`ObservableObject`实现`INotifyPropertyChanged`。
- 使用`RelayCommand`实现命令绑定，避免代码behind。
- 视图模型之间通过消息传递（`IMessenger`）进行通信（如“会话切换消息”“保存消息”）。

### 5.4 性能优化要求
- **启动优化**：使用异步初始化，非核心服务延迟加载，确保启动时间<3秒。
- **UI虚拟化**：会话列表、章节列表使用`VirtualizingStackPanel`，实现UI虚拟化，支持大量数据流畅滚动。
- **异步操作**：所有API调用、数据库操作、文件操作使用`async/await`，避免阻塞UI线程。
- **内存管理**：及时释放大对象（如长章节内容），使用弱事件（`WeakReferenceMessenger`）避免内存泄漏。
