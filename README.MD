## 项目结构

### 1. 模型结构

模型结构如下：
1. 提问端：接收用户的问题，并调用模型进行问题解析和提炼；
2. 收集端：接收提问端模型返回的结果，分析调用哪个prompt，并调用模型进行结果收集；
3. 幻想端：接收收集模型返回的结果结合记忆，并调用模型进行结果幻想，生成故事，返回结果给用户；
4. 总结端：接受幻想端的结果储存进入记忆；
5. 记忆容器：储存处理记忆的工具；
6. io工具：提供io接口将数据连接到本地端和远程端；
7. 中心处理：中心调用模块
8. 工具：提供各种问题的解决办法，提供无用重复单元的封装；

#### 1. 提问端结构

职责：语义分析；

提问端结构如下：
1. 提问模块：接收用户输入，调用约束解析；
2. 模型调用模块：调用模型进行问题解析和提炼；
3. 模型处理模块：处理模型返回的结果；
4. 存储模块：存储到模型容器中；

#### 2. 收集端结构

职责：收集资料；

收集端结构如下：
1. 模型调用模块：调用模型进行结果收集；
2. 调用模块：调用模型进行结果收集，返回数据并分成prompt；
3. 处理模块：处理模型返回的结果，将结果进行分类；
4. 存储模块：存储到模型容器中；

#### 3. 幻想端结构

职责：生成故事；

幻想端结构如下：
1. 模型调用模块：调用模型进行结果幻想，生成故事；
2. 处理模块：处理模型返回的结果，将结果进行分类；
3. 存储模块：存储到模型容器中；

#### 4. 总结端结构

职责：总结结果；

总结端结构如下：

1. 模型调用模块：调用语义分析模块模型进行结果总结；
2. 处理模块：处理模型返回的结果，将结果进行分类；
3. 存储模块：存储到模型容器中；

#### 5. 记忆容器结构 

职责：存储记忆,处理最终结果，保持在一次创作到结束时，不会发生文章偏离主线；

记忆容器结构如下：
1. 存储模块：存储到模型容器中；
2. 读取模块：从模型容器中读取数据；
3. 处理模块：处理模型返回的结果，将结果进行分类；
