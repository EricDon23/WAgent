## 一、系统概述

### 1.1 什么是 WAgent？

WAgent V3.1 是一个基于公开大模型实现的**多AI协同智能写作系统**，采用三角色流水线协作架构（导演AI → 研究员AI → 作家AI），模拟人脑的复杂创作过程。

**V3.1 核心升级**：
- ✓ **Redis双向同步**：启动后第一个执行的步骤，确保数据一致性
- ✓ **分层存储**：global.json（锁定总设定）+ chapters/{x}.json（章节设定）
- ✓ **差异化生成**：第一章生成全局设定，后续章节仅生成章节内容
- ✓ **资料库优先**：研究AI先查本地库，缺失再联网搜索
- ✓ **完整数据加载**：作家AI生成前加载所有可用数据

### 1.2 核心特性

| 特性 | 说明 |
|------|------|
| **三AI协同创作** | 导演AI(豆包) + 研究员AI(通义千问) + 作家AI(DeepSeek) |
| **Redis双向同步** | 本地↔Redis全量同步，本地文件绝对优先 |
| **分层存储机制** | global.json（总设定锁定）+ chapters/{x}.json（章节独立） |
| **防吃书检测** | 非第一章强制加载总大纲+前章上下文，禁止凭空生成 |
| **10阶段进度条** | 实时显示AI思考过程，无黑盒等待 |
| **rich库美化** | Panel/Table/颜色区分，界面整洁美观 |
| **统一入口** | main.py为唯一入口，WAgenter.bat一键启动 |

### 1.3 三AI协作架构（V3.1）

```
用户启动 WAgenter.bat
    ↓
[步骤1] 环境检查 → [步骤2] 数据双向同步（第一个执行步骤）
    ↓
[步骤3] 存储服务检查 → [步骤4] AI引擎就绪
    ↓
进入控制塔主菜单
    ↓
选择/新建会话 → 进入对话模式
    ↓
输入创作需求
    ↓
[导演AI] → 分层生成设定（第一章: global.json+chapters/1.json）
         → （后续章: 仅chapters/{x}.json + 追加到global.json）
    ↓
用户确认设定
    ↓
[研究AI] → 先查db/info_db.json（本地资料库）
         → 缺失部分联网检索
         → 同时写入总资料库 + 章节资料库
    ↓
[作家AI] → 加载所有数据（总设定+总大纲+当前章+资料库）
         → 生成章节正文（2000-6000字）
         → 显示10阶段进度 + 实时思考流
    ↓
存储 → 更新Redis + 本地文件 + 进度
    ↓
用户反馈 → 满意(下一章) / 不满意(重新生成) / 完成
```

## 二、快速开始

### 2.1 环境要求

- **Python**: 3.10+ (推荐3.10+)
- **操作系统**: Windows/Linux/macOS
- **网络**: 需要访问AI API（豆包/通义千问/DeepSeek）
- **Redis**: 可选（未安装自动降级为纯本地模式）

### 2.2 安装步骤

```bash
# 1. 进入项目目录
cd WAgent

# 2. 创建虚拟环境（推荐）
python -m venv venv
venv\Scripts\activate          # Windows
source venv/bin/activate        # Linux/macOS

# 3. 安装依赖
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 4. 配置环境变量
cp .env.example .env
# 编辑 .env 文件，填入API密钥

# 5. 启动系统（双击或命令行）
WAgenter.bat
# 或
python main.py
```

### 2.3 启动方式

```bash
# 方式一：双击启动（推荐）
WAgenter.bat

# 方式二：控制塔模式（默认）
python main.py --tower

# 方式三：跳过同步直接启动
python main.py --no-sync

# 方式四：环境检查
python main.py --check

# 方式五：手动触发同步
python main.py --sync
```

## 四、核心功能详解

### 4.1 Redis双向同步（V3.1核心）

**实现位置**: `sync/sync_manager.py`

**强制要求**（错误修改.md）：
- 程序启动后**第一个执行**的步骤
- 完成前**禁止任何创作操作**
- 本地`stories/` ↔ Redis 全量双向同步
- 冲突处理：**本地文件绝对优先**
- 同步前自动备份到`backups/sync_YYYYMMDD/`
- 失败自动回滚

**同步过程可视化**：
```
  ============================================================
    WAgent V3.1 数据双向同步
  ============================================================

  [1/4] 检查存储服务...
    ✓ Redis可用 - 将执行双向同步

  [2/4] 创建同步前备份...
    ✓ 备份完成: backups/sync_20260419_210000

  [3/4] 收集同步项目...
    发现 33 个待同步项

  [4/4] 执行双向同步...
    --------------------------------------------------
    进度: [100%] (33/33) - 完成

    同步统计:
      • 上传到Redis: 12 个
      • 从Redis下载: 0 个
      • 跳过（本地优先）: 21 个
      • 失败: 0 个
      • 耗时: 1.23 秒

    ✓ 双向同步完成！
  ============================================================
```

### 4.2 分层存储机制（V3.1核心）

**实现位置**: `storage/local.py`

**强制要求**（错误修改.md）：
- `stories/{story_id}/global.json`：**总设定、总大纲、总人物关系**（锁定，AI不能自动覆盖）
- `stories/{story_id}/chapters/{x}.json`：**第x章大纲、人物、设定**
- 非第一章生成前，**必须强制加载** global.json + 前一章上下文

**存储结构示例**：
```
stories/
└── story_20260419_164634/
    ├── global.json           # 总设定（锁定）
    │   {
    │     "story_name": "星际迷途",
    │     "theme": "太空探险",
    │     "characters": [...],
    │     "plot_outline": [...],
    │     "total_chapters": 3
    │   }
    ├── chapters/
    │   ├── 1.json            # 第1章设定
    │   ├── 2.json            # 第2章设定
    │   └── 3.json            # 第3章设定
    ├── drafts/
    │   ├── chapter_1.md      # 第1章正文
    │   ├── chapter_2.md      # 第2章正文
    │   └── chapter_3.md      # 第3章正文
    ├── research/
    │   └── research_xxx.json # 研究资料
    └── state.json           # 进度状态
```

### 4.3 导演AI（DirectorAI）

**实现位置**: `ai/director.py`

**模型配置**:
- 模型: doubao-seed-2-0-pro-260215（豆包）
- 温度: 0（输出稳定一致）
- 用途: 生成结构化故事设定

**差异化输出规则**（错误修改.md强制要求）：

| 场景 | 输出位置 | 内容 |
|------|----------|------|
| **第一章** | global.json + chapters/1.json | 总标题、总设定、三幕式总大纲、总人物关系 + 第1章标题/大纲/人物/主题 |
| **后续章节** | chapters/{x}.json + 追加到global.json | 第x章标题、大纲、人物、主题（自动追加到总大纲和人物关系） |

**生成后必须显示**（6项信息卡片）：
1. 标题
2. 人物列表
3. 主题
4. 大纲
5. 字数
6. 章节号

### 4.4 研究员AI（ResearcherAI）

**实现位置**: `ai/researcher.py`

**模型配置**:
- 模型: qwen-plus（通义千问）
- 温度: 0（资料客观准确）
- 用途: 联网检索创作所需资料

**资料查询顺序**（错误修改.md强制要求）：
1. **先查本地总资料库** `db/info_db.json`
2. **再查网络**（MCP百度搜索服务）
3. **同时写入**：
   - 总资料库 `db/info_db.json`（去重）
   - 章节资料库 `stories/{story_id}/research/chapter_{x}.json`
4. 资料数量严格控制在 **5-20个**

### 4.5 作家AI（WriterAI）

**实现位置**: `ai/writer.py`

**模型配置**:
- 模型: deepseek-chat（DeepSeek）
- 温度: 1（允许创造性发挥）
- 用途: 生成小说章节正文

**强制数据加载**（错误修改.md要求）：
生成前必须加载**所有可用数据**：
- 总设定、总大纲、总人物关系（来自global.json）
- 当前章大纲、当前章人物、当前章设定（来自chapters/{x}.json）
- 总资料库、当前章资料库（来自db/ + research/）

**输出约束**：
- 字数: 2000-6000字/章
- 格式: Markdown
- 校验: 章节标题序号100%一致

## 五、技术架构

### 5.1 五层架构体系（V3.1标准）

```
交互层：CLI控制塔（rich库美化）
    ↓
智能体层：WAgentMain统一调度
    ↓
AI处理层（三AI流水线）
├─ 导演AI (ai/director.py)
├─ 研究员AI (ai/researcher.py)
└─ 作家AI (ai/writer.py)
    ↓
工具层
├─ 存储引擎 (storage/)
│   ├── local.py（本地文件存储）
│   └── redis.py（Redis高速缓存）
├─ 同步管理 (sync/sync_manager.py)
└─ 界面美化 (ui/cli.py)
    ↓
基础支撑层
├─ stories/（故事存储）
├─ db/（通用资料库）
├─ backups/（自动备份）
└─ logs/（运行日志）
```

### 5.2 技术栈

| 技术/工具 | 版本 | 用途 |
|----------|------|------|
| Python | 3.10+ | 系统核心运行环境 |
| LangChain | 0.2.17+ | AI Agent框架 |
| LangChain OpenAI | 0.1.22+ | 大模型接口适配 |
| Pydantic | 2.10.0+ | 结构化输出校验 |
| Redis | 7.2+ | 高速缓存与持久化 |
| rich | latest | 控制台UI美化 |
| python-dotenv | 1.0.1+ | 环境变量管理 |

### 5.3 V3.1 vs V2.1 架构变更

| 变更项 | V2.1 | V3.1 |
|--------|------|------|
| **主程序入口** | control_tower.py + 多个独立文件 | **main.py（唯一入口）** |
| **数据同步** | GModule手动调用 | **SyncManager（启动后第一个执行）** |
| **存储结构** | 单一JSON文件 | **分层存储（global.json + chapters/）** |
| **导演AI输出** | 统一格式 | **差异化（第一章 vs 后续章节）** |
| **研究AI策略** | 直接联网 | **先本地库 → 再网络** |
| **作家AI数据加载** | 部分加载 | **完整加载所有可用数据** |
| **进度显示** | 无 | **10阶段进度条 + 实时思考流** |
| **界面美化** | 纯文本 | **rich库Panel/Table/颜色** |
| **项目结构** | data/ + tools/ + tests/ | **标准结构（删除冗余）** |

## 附录：常见问题

**Q: 启动时报"找不到模块"错误？**
A: 确保已安装依赖 `pip install -r requirements.txt`，并在项目根目录运行。

**Q: Redis连接失败怎么办？**
A: 系统自动降级为纯本地模式，功能不受影响。建议生产环境部署Redis。

**Q: 如何查看同步状态？**
A: 启动时会自动显示同步统计（上传X个/下载X个/跳过X个）。也可手动执行 `python main.py --sync`。

**Q: 生成的故事在哪里？**
A: `stories/{story_id}/drafts/chapter_{n}.md`，每章一个Markdown文件。

**Q: 如何重置所有数据？**
A: 删除 `stories/` 目录下所有子目录即可。Redis数据会在72小时后自动过期。
